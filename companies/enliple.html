🚀 추천 데이터 송출 API 성능 개선

📅 2021.03 - 2021.07 👥 추천 팀 리더 🎯 API 설계 및 개발 / 팀 리딩

프로젝트 요약:

  • 추천 배치에서 생성된 다양한 종류의 추천 데이터를 조합하여 송출 서버에 제공하는 REST API 개발
  • 광고 노출 시 응답 속도와 추천의 질이 중요한 환경에서, Spring WebFlux를 활용한 비동기 병렬 호출 API 개발

프로젝트 상세:

  • 웹플럭스를 처음 도입하는 상황에서 레퍼런스 부족으로 새로운 기술 도입에 망설임이 있었지만, 응답 속도 최적화를 위한 새로운 기술이 필요하다는 판단하에 팀장으로서 팀원들과 함께 웹플럭스 도입 결정
  • RDB를 사용하고 있어 논블로킹 방식 유지가 어려웠지만, 웹플럭스 도입만으로도 이벤트 루프 모델로 동작하여 사용자 요청이 빈번하지만 CPU 사용량이 낮은 작업의 API에서 효과 확인
  • 논블로킹을 유지하기 위한 설계 구상을 계획하며, 지속적인 기술 개선을 향한 노력 진행

역할 및 성과:

  • 추천 팀 리더로 팀원들과 협업하여 프로젝트 설계 및 개발 담당
  • ElasticSearch의 유사도 기반 데이터 조회 시 빈번한 호출을 고려하여 Redis 도입으로 응답 속도 향상
  • Spring WebFlux 활용하여 비동기 처리를 유지하며 응답 속도 최적화
  • 약 200만 동시 접속자 발생 시 WebFlux의 성능 우위 확인

사용 기술

Java Spring WebFlux Redis ElasticSearch MongoDB

프로젝트 상세:

  • 사내에서 수집한 사용자 구매 로그 데이터를 카프카 서비스를 활용하여 컨슘
  • IBCF(아이템 베이스 기반) 추천 알고리즘을 사용한 추천 배치 서비스의 고도화 및 튜닝
  • 대량의 로그와 데이터로 인한 메모리 부족(OOM) 문제로 동작하지 않는 배치를 정상적으로 처리하는 프로젝트
  • 기존 코드의 분석을 통해 문제점과 메모리 누수 확인

  • 기존 레거시 코드의 문제점을 파악하고, 카프카 컨슈머 서비스, 배치 서비스, API 서비스 등의 분석 및 개발 담당
  • 동작하지 않던 배치를 정상 동작으로 복구함으로써 새로운 추천 데이터 생성 가능
  • 배치 완료 시간을 12시간에서 3시간으로 단축하여 새로운 로그에 대한 실시간성 추천 데이터 생성 성능 향상
