라이브 트래픽 사전 증설

경기 시작 전 Airflow가 자동으로 ECS 서비스를 증설하고, 종료 후 축소. 운영팀이 경기별 등급을 설정하면 그에 맞게 동적 스케일링.

sequenceDiagram
    participant GAME as 경기 정보 API
    participant AF as Airflow
    participant ECS as ECS Cluster
    participant SLACK as Slack

    GAME->>AF: 경기 일정 동기화
    Note over AF: 경기 1시간 전 트리거
    AF->>ECS: 등급별 증설 요청
    ECS-->>AF: 증설 완료
    AF->>SLACK: 증설 완료 알림
    Note over ECS: 경기 진행 중
    Note over AF: 경기 종료 감지
    AF->>ECS: 스케일 다운
    AF->>SLACK: 축소 완료 알림

멀티 서비스 인프라 분리

NOW, GOLF, NOW JP가 각각 독립된 AWS 계정으로 운영. GOLF는 신규 프로젝트라 90% 이상 Terraform으로 관리 중.

graph TB
    IC[AWS Identity Center] --> NOW[SPOTV NOW\nAWS Account]
    IC --> GOLF[SPOTV GOLF\nAWS Account]
    IC --> JP[SPOTV NOW JP\nAWS Account]

    NOW --> NOW_ECS[ECS + EC2\n콘솔 + Terraform 혼용]
    GOLF --> GOLF_ECS[ECS\nTerraform 90%+]
    JP --> JP_ECS[ECS\n별도 인프라]
GOLF 프로젝트는 처음부터 Terraform으로 인프라를 구성하여 코드 기반 관리. NOW는 기존 콘솔 기반 리소스와 Terraform이 혼용된 상태.
