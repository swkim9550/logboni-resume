🔧 라이브 스트리밍 대응 서버 사전 증설 자동화 시스템

📅 2024.09 - 2024.10

라이브 스포츠 스트리밍 특성상 특정 경기(손흥민 경기 등)에서 급격한 트래픽 증가에 대비한 사전 서버 증설 자동화 시스템 구축

프로젝트 배경

  • 손흥민 경기 등 인기 경기 시 20만~30만 동시 접속자 발생
  • 일반적인 Auto Scaling은 반응 시간이 있어 급격한 트래픽 증가에 대응 한계
  • 경기 시작 전 약 1시간 전부터 서버를 미리 증설하여 "서버 워밍업" 필요

주요 구현 사항

  • Airflow 기반 스케줄링: 경기 시간 기준 1시간 전 자동 서버 증설 스크립트 실행
  • 운영팀 친화적 UI 구축: 특정 경기별로 High/Medium/SuperHigh 등급 설정
  • 등급별 수용 인원 사전 정의: 각 등급에 따른 서버 증설 규모 자동 결정
  • Shell Script 자동화: AWS EC2 인스턴스 생성 및 로드밸런서 등록 자동화
  • 경기 종료 후 자동 스케일 다운으로 비용 최적화

성과

  • 대규모 라이브 이벤트 시 서버 다운 없이 안정적 서비스 제공
  • 운영팀의 수동 서버 관리 업무 완전 자동화
  • 트래픽 급증 상황에서도 사용자 경험 품질 유지

사용 기술

Apache Airflow AWS EC2 Shell Script Application Load Balancer CloudWatch Auto Scaling 운영 UI
